BLAK KEYMAPS                                            *blak-keymaps*

The rule: common operations get memorable mappings; uncommon operations get
commands and pickers.

User mappings from lua/blak/user.lua use the same registry. Add entries to
keymaps with key, action, and description. action may be a command string or Lua
function, and mode may be one mode or a list. Use disable = true to disable a
Blak mapping.

FIND
  <Space><Space>   Smart file find
  <Space>/         Grep
  <Space>ff        Find files
  <Space>fg        Grep
  <Space>fb        Buffers
  <Space>fr        Recent files
  <Space>fc        Commands
  <Space>fk        Keymaps
  <Space>fh        Help

SAVE
  <C-s>            Save, in modes where Neovim leaves it free
  <D-s>            Save, where Command-s is forwarded by the UI

BUFFERS
  <Space>`         Toggle last file
  <Space>bd        Delete buffer
  <Space>bn        Next buffer
  <Space>bp        Previous buffer

WINDOWS
  <Space>ws        Split window below
  <Space>wv        Split window right
  These use native :rightbelow split and :rightbelow vsplit, and only bind if
  the keys are free.

EXPLORER
  <Space>e         Configured explorer (Oil by default)
  Blak leaves Neovim's native - motion alone.

TERMINAL
  <Space>tt        Configured terminal provider
  Set terminal.toggle_key in lua/blak/user.lua to change this mapping.

CUSTOM
  lua/blak/user.lua can add or override mappings:
>
  return {
    keymaps = {
      { key = "<leader>sg", action = "<cmd>BlakPick grep<cr>", description = "Grep" },
      {
        mode = { "n", "x" },
        key = "<leader>y",
        action = '"+y',
        description = "Yank to clipboard",
      },
      {
        key = "<leader>rn",
        action = function()
          vim.lsp.buf.rename()
        end,
        description = "Rename symbol",
      },
      { key = "<leader>/", disable = true },
    },
  }
<
  Active custom mappings require description so they appear in :BlakKeys.

GIT
  [h               Previous git hunk
  ]h               Next git hunk
  <Space>gs        Stage hunk
  <Space>gr        Reset hunk
  <Space>gS        Stage buffer
  <Space>gR        Reset buffer
  <Space>gp        Preview hunk
  <Space>gb        Blame line
  <Space>gd        Diff this

CODE
  gd               Definition
  gD               Declaration
  gr               References
  K                Hover
  <Space>ca        Code action
  <Space>cr        Rename
  <Space>cf        Format

DIAGNOSTICS
  [d               Previous diagnostic
  ]d               Next diagnostic
  <Space>xx        Diagnostics picker
  <Space>xd        Line diagnostic

BLAK
  <Space>?         Show Blak keymaps
  <Space>ll        Lazy
  <Space>lo        Blak overview
  <Space>lc        Blak config
  <Space>lu        Blak update
  <Space>lU        Blak upgrade
  <Space>lr        Blak rollback
  <Space>ld        Blak doctor
  <Space>le        Blak extras
  <Space>lk        Blak keymaps
  <Space>ln        Blak news
  <Space>ls        Blak splash
  <Space>lt        Install Blak tools
  <Space>lT        Install Treesitter parsers

EXTRAS
  debug.dap maps <Space>d* debug actions: breakpoints, continue, step, REPL,
  terminate, and dap-ui.
  test.neotest maps <Space>T* test actions: nearest, file, debug, summary, and
  output.
  editor.aerial maps <Space>co to toggle the code outline.
  editor.overseer maps <Space>oo, <Space>or, and <Space>oq for task workflows.
  editor.refactoring maps <Space>r* refactor and print-debug actions.
  editor.render-markdown maps <Space>um to toggle Markdown rendering.
  editor.todo-comments maps ]t, [t, and <Space>xT for TODO comments.
  editor.trouble maps <Space>xX, <Space>xQ, <Space>xL, <Space>cO, and
  <Space>cR for Trouble lists.
  editor.harpoon maps <Space>ha, <Space>hh, <Space>hp, <Space>hn, and
  <Space>h1 through <Space>h4 for Harpoon file marks.
  ai.supermaven maps <Space>aS to toggle Supermaven, <M-l> to accept an inline
  suggestion, <M-w> to accept one word, and <M-]> to clear the suggestion.
  editor.window-navigation maps <C-h>, <C-j>, <C-k>, and <C-l> to move
  between windows.
  ui.comfy-line-numbers maps displayed labels with j/k and Up/Down motions.
  Example: 11j and 11<Down> both use the comfy label 11.
  ai.claudecode maps <Space>a* Claude Code actions: toggle, focus, resume,
  continue, select model, add buffer, send selection, and accept or deny diffs.
