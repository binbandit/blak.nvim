*blak.txt*  Blak Neovim distribution

BLAK                                                     *blak*

Blak is a native-first Neovim distribution built around a black-hole
identity and a low-bloat contract.

COMMANDS                                                 *blak-commands*

:Blak
    Open a short overview.

:BlakDoctor
    Run |:checkhealth| for Blak.

:BlakKeys
    Show keymaps registered by Blak core, enabled extras, and user.lua.

:BlakNews
    Open the Blak release notes.

:BlakDocs
    Open the Blak docs site at https://getblak.dev/start/why/.

:BlakConfig
    Open lua/blak/user.lua. If it does not exist, create it from
    lua/blak/user.example.lua first.

:BlakPick {kind}
    Run a picker. Kinds include: files, grep, buffers, recent, commands,
    keymaps, help, diagnostics, lsp_symbols, workspace_symbols.

:BlakExtras [list|enable|disable|sync] [id]
    Manage optional extras. With no args or "list", opens the extras UI.
    See |blak-extras|.

:BlackExtras [list|enable|disable|sync] [id]
    Alias for :BlakExtras.

:BlakUpdate
    Create a rollback snapshot and run |:Lazy update| within the current
    channel. Pending breaking migrations block this command.

:BlakUpgrade
    Create a rollback snapshot, apply pending upgrade migrations, then run an
    intentional bigger update.

:BlakRollback
    Restore the latest rollback snapshot and run |:Lazy restore|.

:BlakToolsInstall
    Install Mason tools required by enabled extras.

:BlakTreesitterInstall
    Install configured Treesitter parsers.

:BlakTerminal [cmd]
    Toggle the configured terminal provider. Native terminal splits are the
    default. With [cmd], run that command instead of the user's shell.

:BlakFormat
    Format the current buffer with Conform and the configured LSP fallback.

:BlakFormatToggle[!]
    Toggle format-on-save for the current buffer. Use bang for global.

:BlakSplash
    Preview the black-hole splash animation.

See |blak-keymaps| for the matching <Leader>l shortcuts.

CONFIGURATION                                            *blak-config*

Run >
  :BlakConfig
<
or copy >
  lua/blak/user.example.lua
< to >
  lua/blak/user.lua
< and return a table of overrides, or a function for full Lua control.

For example, to use SuperTab-style completion where Blak can delegate to the
completion engine: >
  return { completion = { super_tab = true } }
<
To make the active colorscheme use transparent editor backgrounds: >
  return { ui = { transparent = true } }
<
Theme-native setup options can be passed through ui.theme when the
colorscheme exposes a Lua setup() function: >
  return { ui = { colorscheme = "catppuccin-mocha", theme = { flavour = "mocha" } } }
<
Personal lazy.nvim specs can be added under plugins.specs: >
  return {
    plugins = {
      specs = {
        { "folke/trouble.nvim", cmd = "Trouble", opts = {} },
      },
    },
  }
<
For full Lua control, return a function. Blak passes the config table it is
building before validation and extras apply: >
  return function(config)
    config.picker.provider = "snacks"
    table.insert(config.extras.enabled, "lang.typescript")
  end
<
Use hooks.after for Lua that should run after Blak setup and after successful
user.lua reloads.

Saving lua/blak/user.lua while Blak is running reloads the merged config and
refreshes the current session. Run :BlakExtras sync or :Lazy sync after adding
or removing plugin-backed extras; restart to unload runtime hooks that already
ran.
